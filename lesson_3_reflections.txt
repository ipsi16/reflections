When would you want to use a remote repository rather than keeping all your work local?
  When multiple people will be working collectively on different features of the project in the repository. 
  The remote repository will store the changes made by each contributor and enable sharing of code. 

Why might you want to always pull changes manually rather than having Git automatically stay up-to-date with your remote repository?
	Sometimes the changes incorporated in the remote may be unwanted. 
	Pulling changes manually gives us the the power to control the changes made to our local repository in case of any changes made in the local
